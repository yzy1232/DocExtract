# 前端使用文档

本文档面向系统使用者与实施人员，说明如何在前端页面完成从模板配置到结果导出的完整流程。

## 1. 页面入口

登录成功后，你可以通过以下入口访问功能模块：

- 左侧导航：工作台、文档管理、模板管理、提取任务、使用文档（管理员可见系统配置）
- 工作台快捷操作：上传文档、创建模板、新建提取、查看结果、使用文档
- 使用文档页面路径：`/guide`

## 2. 标准操作流程

建议按以下顺序操作，以减少重复返工：

### 步骤 1：创建模板

入口：模板管理 -> 创建模板 / 智能推断模板

建议：

- **智能推断**：支持选择已解析的样板文档，系统会自动通过 LLM 与规则混合引擎生成名称、描述以及所有字段的属性与必填项，可在此基础上人工修改保存。
- 字段名称使用业务可读词汇
- 字段描述写清楚来源位置、业务口径和单位
- 必填字段只保留真正必须项，避免过度约束

### 步骤 2：上传文档

入口：文档管理 -> 上传文档

支持格式：PDF、DOCX、XLSX、TXT

建议：

- 扫描件尽量保证清晰度
- 同批次文档可统一命名，便于追踪
- **文档预览**：文档上传与处理完成后，可点击“在线预览”原生查看原文档的保留排版（如 PDF/图片）或通过 HTML 转换进行降级预览（Word/Excel）。

### 步骤 3：发起提取任务

入口：提取任务 -> 创建提取任务

操作要点：

- 选择目标文档
- 选择匹配模板
- 提交任务后在任务列表查看进度
- **任务状态**：大文件处理时长可能较久。可在处理期间操作**“取消任务”**及时中止正在执行和排队的作业。

### 步骤 4：查看提取结果

入口：提取任务 -> 任务详情/提取结果

重点关注：

- 字段值是否符合业务语义
- 置信度较低字段是否需要人工复核

### 步骤 5：导出结果

导出格式：Excel、JSON、CSV

建议：

- 对外系统对接优先使用 JSON
- 人工核验和表格流转优先使用 Excel

## 3. 角色分工建议

- 业务人员：持续优化模板字段描述
- 审核人员：优先复核低置信度字段
- 管理员：维护模型配置、查看系统状态与任务队列

## 4. 常见问题

### 4.1 上传后不能立即提取

原因：文档处于异步解析阶段。

处理：在文档列表确认状态可用后再发起提取。

### 4.2 提取结果为空或不稳定

原因：模板字段定义过于模糊，或文档内容质量较低。

处理：细化字段描述并重新执行提取任务。

### 4.3 没有系统配置菜单

原因：当前账号非管理员角色。

处理：联系管理员分配对应权限。

### 4.4 系统提示数据库丢失或 Redis 异常断线

原因：底层集群出现核心数据脱机或误删。

处理：在登录页如遇 5xx 不可用系统可根据报错重定向，并联系运维借助应急配置执行系统一键恢复与容灾检测流程。
### 5. 运维联调建议

- 首次部署后先在工作台检查系统健康状态
- 定期检查任务积压情况，确保 Worker 在线
- 在切换 LLM 提供商后，先做一轮小样本模板验证
